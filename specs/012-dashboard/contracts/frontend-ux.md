# Contract — Frontend UX & Component Map (built with `/ui-ux-pro-max`)

The React SPA's visual + interaction contract. The design system below is generated by the
**`/ui-ux-pro-max`** skill (design-system query: *security operations / incident response / dark mode*)
and is the source of truth for the look; deviations per page are recorded here. Realizes FR-022 (clean,
modern, responsive, demo-ready) and binds every other dashboard FR to a concrete surface.

---

## Design system (from ui-ux-pro-max — RD9)

**Style**: Dark Mode (OLED) — professional, trustworthy security-tooling aesthetic; WCAG-AAA-capable
contrast; excellent performance.

**Color tokens** (Tailwind):
| Role | Hex | Tailwind |
|------|-----|----------|
| Background | `#020617` | slate-950 |
| Surface / card | `#0F172A` | slate-900 |
| Surface raised / border | `#1E293B` | slate-800 |
| Text | `#F8FAFC` | slate-50 |
| Muted text | `#94A3B8` | slate-400 |
| Primary action / positive | `#22C55E` | green-500 |

**Semantic colors** (layered for SOC meaning — always paired with a **label/icon**, never color alone):
- Severity: low `#38BDF8` (sky), medium `#FACC15` (amber), high `#FB923C` (orange), critical
  `#EF4444` (red).
- Status/disposition badges: in-flight (slate/blue), awaiting-approval (amber, attention), resolved /
  remediated (green), escalated (orange), rejected/expired (red/slate).

**Typography**: **Fira Sans** (body/UI) + **Fira Code** (mono — incident IDs, IPs, hashes, tokens,
trace span names, JSON evidence). `line-height` 1.5–1.75; body ≥ 14px UI / 16px prose.

**Layout**: **Application shell** — fixed left **sidebar** (Queue · KPIs · sign-out + connection
indicator), top **bar** (incident title/breadcrumb on drill-down, operator menu), scrollable **main**.
The matcher's "Horizontal Scroll Journey" landing pattern is **rejected** — a dense ops console uses a
stable shell, not a marketing scroll narrative.

**Charts** (Recharts): line/area = volume-over-time; horizontal bar (or donut) = disposition split;
stat cards = MTTD + memory-hit %; custom collapsible tree = trace inspector.

**Effects**: subtle (150–300ms color/opacity transitions, visible focus rings, minimal glow on key
metrics). No layout-shifting hover; `transform`/`opacity` only; honor `prefers-reduced-motion`.

**Non-negotiable a11y** (ui-ux-pro-max checklist): SVG icons only (Lucide — no emoji); `cursor-pointer`
on all clickable; focus rings; color never the sole signal; ≥44px touch targets; alt/aria on icon
buttons; labels tied to inputs; responsive at 375/768/1024/1440.

## Key interaction contracts

- **Queue (US1)**: server-pushed updates via SSE (FR-023) patch the table; filter by status (incl.
  `awaiting_approval`) + severity; toggle active/resolved/all; paginate; row click → detail. Empty state
  when no incidents. Loads < 2s for 200 rows (SC-009); changes appear ≤5s without refresh (SC-010).
- **Approval (US2)**: the panel renders the **specific** proposed destructive actions + rationale +
  **live countdown** to `deadline_at` **before** any decision can be submitted (SC-002). Approve/Reject
  call `POST /approvals/{id}/decision`; buttons disable during the async call (loading state). If the
  server returns `409` (already decided / expired by the sweeper or another tab), the UI shows
  "already decided / expired" and disables the controls — **no second remediation** (FR-012, SC-008).
  Post-decision disposition reflected ≤3s (SC-003) via the response + an SSE delta.
- **Trace (US3)**: collapsible triage→enrichment→response tree; each node shows tokens in/out, model,
  latency, status; **missing tokens render "unknown", not 0** (FR-015); error spans are clearly marked
  with the recovery visible (FR-016). A seeded secret never appears (FR-017, SC-004). Mid-pipeline
  incident → graceful partial/empty tree, not an error.
- **KPIs (US4)**: four metric families render as visualizations (not raw tables) and reconcile with the
  records (SC-006). Memory-hit shown as `hits/enriched` with the % and denominator visible.
- **Global resilience**: backend down/timeout → ErrorState with retry (FR-021); stream drop →
  reconnecting indicator + refetch fallback (FR-023); stale session → next action routes to `/login`
  (FR-003).

## Stack & build

Vite + React 18 + TS + Tailwind + shadcn/ui + TanStack Query + TanStack Table + Recharts + React Router
+ Lucide (RD1). Token in memory + `sessionStorage`; `Authorization: Bearer` on REST, `?token=` on SSE.
Built by `deploy/frontend/Dockerfile` (node build → nginx static + reverse-proxy), `frontend` service in
`compose.yaml` (RD10).

## Tests (frontend)

Vitest + React Testing Library: queue render/filter/paginate; approval guard (409 → "already decided");
trace render incl. "unknown" telemetry + marked error span + no-secret assertion; empty/error/reconnect
states. One Playwright e2e: sign-in → queue → open incident → approve → see `remediated`.
